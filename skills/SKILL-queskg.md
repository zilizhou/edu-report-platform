# 作业与试卷题目知识点标注技能

## 概述

本技能面向作业、试卷、练习册等题目集合类文档，通过逐题理解与分析，为每道题目标注一个或多个对应知识点，最终输出结构化 CSV 文件。产出结果可用于：题库建设、考点统计、学情分析、知识薄弱点诊断、自适应学习系统等场景。

---

## 工作流程

### 第一步：读取题目文档

根据文件类型选择对应读取方式：

| 文件类型 | 处理方式 |
|---------|---------|
| `.pdf` | 参考 `/mnt/skills/public/pdf-reading/SKILL.md` |
| `.docx` | 参考 `/mnt/skills/public/docx/SKILL.md` |
| `.pptx` | 参考 `/mnt/skills/public/pptx/SKILL.md` |
| `.xlsx` / `.csv` | 参考 `/mnt/skills/public/xlsx/SKILL.md`；题目通常在某一列中 |
| 纯文本/Markdown | 直接读取，按题号或空行切分 |

> **提示**：如果用户直接粘贴了题目文本，跳过本步骤，从第二步开始处理。

**读取后确认：**
- 识别文档中的**题目总数**
- 识别**题型结构**（如：单选题 10 题、填空题 5 题、大题 3 题）
- 确认**是否已有参考答案**（有助于提升知识点标注准确性）
- 如发现文档结构复杂（图文混排、表格嵌套等），告知用户可能影响识别精度

---

### 第二步：题目解析与结构化

在标注知识点之前，先将每道题目解析为结构化记录。

**题目识别规则：**
1. 以题号为切分依据（如：`1.`、`（一）`、`第1题`、`Q1`、`一、`等格式）
2. 大题（如"三、解答题"）下的每个子题单独记录，编号格式为 `3-1`、`3-2`……
3. 题干、选项、分值尽量分开提取
4. 若题目含图表或公式无法提取，在 `note` 字段注明"含图片/公式"

**题型分类（`question_type` 字段）：**

| 题型标签 | 说明 |
|---------|------|
| `single_choice` | 单项选择题 |
| `multiple_choice` | 多项选择题 |
| `true_false` | 判断题 |
| `fill_blank` | 填空题 |
| `short_answer` | 简答题 |
| `calculation` | 计算题 |
| `proof` | 证明题 |
| `essay` | 论述/分析题 |
| `coding` | 编程题 |
| `case_study` | 案例分析题 |
| `other` | 其他 |

---

### 第三步：知识点库准备

标注前需确定使用的知识点来源，按以下优先级处理：

**优先级 1：用户提供了知识点列表**
- 使用用户提供的知识点进行标注（闭集标注）
- 若题目涉及列表外的知识，在 `note` 字段注明，并询问用户是否补充

**优先级 2：用户提供了配套教学资源（大纲/教材）**
- 从资源中提取知识点体系（可联动 `course-knowledge-graph` 技能）
- 以章节知识点为标注范围

**优先级 3：无预设知识点库**
- 根据题目内容自行归纳知识点（开放集标注）
- 知识点命名遵循学科通用术语，粒度适中（避免过于宽泛如"数学"，也避免过细如"第3章第2节例题1"）
- 标注完成后，在统计摘要中列出全部知识点，供用户审核

---

### 第四步：逐题知识点标注

对每道题目进行分析，按以下维度判断：

**标注思路（依次分析）：**

1. **题目考查的核心操作是什么？**
   - 记忆型（定义/概念的复述）→ 对应概念类知识点
   - 理解型（判断、比较、解释）→ 对应原理/关系类知识点
   - 应用型（计算、编程、设计）→ 对应方法/技能类知识点
   - 分析综合型（案例/论述）→ 通常对应多个知识点

2. **涉及哪些核心概念或术语？**
   从题干和选项中识别学科关键词，映射到知识点

3. **解题需要哪些前置知识？**
   如果解题过程需要多步推理，每个关键步骤对应一个知识点

4. **多知识点优先级排序：**
   - `primary`：题目最直接考查的知识点（必填，至少1个）
   - `secondary`：辅助性、背景性知识点（选填，视题目复杂度而定）

**标注数量参考：**

| 题型 | 建议知识点数量 |
|-----|-------------|
| 单选题、判断题 | 1～2 个 |
| 填空题 | 1～3 个 |
| 简答题、计算题 | 2～4 个 |
| 大题（综合/案例/编程）| 3～6 个 |

> **原则**：宁精勿滥。若一道题只考查一个知识点，不必强行添加第二个。

---

### 第五步：难度与认知层次评估（可选）

如用户需要，可同步为每道题目标注：

**难度等级（`difficulty`）：**
- `easy`：直接套用定义或公式，无需推理
- `medium`：需要理解原理或进行多步运算
- `hard`：需要综合多个知识点，或涉及较深推理

**认知层次（`cognitive_level`，参考布鲁姆教育目标分类）：**

| 层次 | 标签 | 说明 |
|-----|------|------|
| 记忆 | `remember` | 回忆事实、定义、公式 |
| 理解 | `understand` | 解释、举例、分类 |
| 应用 | `apply` | 在新情境中使用知识 |
| 分析 | `analyze` | 分解结构、找关联、推断 |
| 评价 | `evaluate` | 作出判断、辩证分析 |
| 创造 | `create` | 设计方案、综合创新 |

---

### 第六步：生成 CSV 文件

#### 主文件：`question_knowledge_tags.csv`

```csv
question_id,question_type,question_stem,knowledge_points,primary_kp,secondary_kp,difficulty,cognitive_level,score,source_file,note
1,single_choice,下列关于TCP协议描述正确的是，TCP协议;可靠传输;三次握手,TCP协议,可靠传输;三次握手,easy,remember,2,期末试卷.pdf,
2,calculation,求函数f(x)=x²+2x的极值，导数;极值求解;函数分析,极值求解,导数;函数分析,medium,apply,5,作业第三章.docx,
3-1,short_answer,简述OSI七层模型各层功能，OSI模型;网络分层;协议栈,OSI模型,网络分层;协议栈,medium,understand,8,期末试卷.pdf,含表格
```

**字段说明：**

| 字段 | 类型 | 是否必填 | 说明 |
|-----|------|---------|------|
| `question_id` | string | ✅ | 题目编号（子题格式：`3-1`）|
| `question_type` | string | ✅ | 题型标签（见第二步分类表）|
| `question_stem` | string | ✅ | 题干文本（超过100字截断，用`…`结尾）|
| `knowledge_points` | string | ✅ | 全部知识点，多个用英文分号 `;` 分隔 |
| `primary_kp` | string | ✅ | 主要知识点（最核心的1个）|
| `secondary_kp` | string | ❌ | 次要知识点，多个用 `;` 分隔，无则留空 |
| `difficulty` | string | ❌ | 难度等级（easy/medium/hard）|
| `cognitive_level` | string | ❌ | 认知层次标签（见第五步）|
| `score` | number | ❌ | 题目分值（若文档中有标注）|
| `source_file` | string | ✅ | 来源文件名 |
| `note` | string | ❌ | 备注（如"含图片"、"公式无法解析"等）|

#### 辅助文件：`knowledge_point_statistics.csv`（知识点统计表）

```csv
knowledge_point,question_count,question_ids,total_score,difficulty_distribution,avg_cognitive_level
TCP协议,3,1;5;12,12,easy:2|medium:1,remember
极值求解,2,2;8,13,medium:2,apply
OSI模型,1,3-1,8,medium:1,understand
```

**字段说明：**

| 字段 | 类型 | 说明 |
|-----|------|------|
| `knowledge_point` | string | 知识点名称 |
| `question_count` | number | 该知识点出现的题目数量 |
| `question_ids` | string | 对应题目编号，用 `;` 分隔 |
| `total_score` | number | 涉及该知识点的题目总分（可为空）|
| `difficulty_distribution` | string | 难度分布，格式：`easy:N\|medium:N\|hard:N` |
| `avg_cognitive_level` | string | 主要认知层次 |

---

### 第七步：质量自检

输出前逐项确认：

- [ ] 每道题目都有至少一个 `primary_kp`，无空值
- [ ] `knowledge_points` 字段包含 `primary_kp` 的内容
- [ ] 多知识点用英文分号 `;` 分隔，无中文顿号或逗号
- [ ] 子题编号格式统一（如 `3-1` 而非 `第3题第1小题`）
- [ ] 题干截断处有 `…` 标记
- [ ] 统计表中知识点名称与主文件完全一致（区分大小写）
- [ ] CSV 编码为 UTF-8，无多余空行

---

## 输出规范

**最终向用户交付：**

1. `question_knowledge_tags.csv` — 题目知识点标注主表（必须）
2. `knowledge_point_statistics.csv` — 知识点频次统计表（推荐）
3. 对话中同步输出**分析摘要**，包含：
   - 共处理题目数量及题型分布
   - 共识别知识点数量
   - 出现频次最高的 Top 5 知识点
   - 知识点覆盖分布（哪些章节/模块考察较多）
   - 特殊情况说明（如无法识别的图片题、公式题数量）

---

## 多文件处理策略

当用户同时提供多份作业或试卷时：

1. **分文件处理**：每份文件独立解析，`source_file` 字段区分来源
2. **题号全局唯一化**：在题号前加文件前缀（如 `exam1_Q1`、`hw2_Q3`）
3. **知识点统一命名**：跨文件若出现同义知识点，统一为同一名称
4. **合并统计**：`knowledge_point_statistics.csv` 汇总所有文件的统计数据

---

## 特殊情况处理

| 情况 | 处理方式 |
|-----|---------|
| 题目含图片（如几何题） | `note` 注明"含图片"，尽量从文字部分推断知识点 |
| 题目含公式无法解析 | `note` 注明"含公式"，根据上下文推断考查内容 |
| 题目语义模糊难以判断知识点 | 标注最可能的知识点，`note` 注明"待核查" |
| 综合大题跨越多个知识域 | 拆分子题分别标注；若无子题则标注所有相关知识点 |
| 同一题目答案涉及多章内容 | 所有章节的核心知识点均列入 `knowledge_points`，取最核心的为 `primary_kp` |
| 用户提供了答案/解析 | 优先参考解析中提到的知识点进行标注，准确性更高 |

## 典型用例示例

**用例1：期末试卷标注**

用户输入："这是我们《计算机网络》的期末试卷，帮我分析每道题考的哪些知识点"

处理步骤：
1. 读取PDF，识别出共40题（单选20、填空10、大题10）
2. 无预设知识点库，开放集标注
3. 逐题分析，如第1题"TCP三次握手"→ 标注 `TCP协议`、`三次握手`、`连接建立`
4. 输出标注表 + 统计摘要（如"TCP/IP协议族"出现12次，是最高频知识点）

**用例2：批量作业知识点统计**

用户输入："这是本学期5次作业，帮我统计每个知识点出现了几次"

处理步骤：
1. 逐份作业读取，题号加文件前缀区分
2. 统一知识点命名（如"导数"和"求导"合并为"导数"）
3. 输出合并统计表，按频次降序排列
4. 在摘要中指出哪些知识点反复出现（可能是重点或难点）

---

## 注意事项

- **保持客观**：知识点标注基于题目内容本身，不主观推测出题意图
- **粒度一致**：同一份文件内知识点粒度要统一，避免混用"函数"（宽泛）和"二次函数图像的对称轴求法"（过细）
- **中英文统一**：同一知识点在全文使用统一语言，不混用（如统一用"梯度下降"或"Gradient Descent"）
- **尊重原题结构**：不合并或拆分原始题目，保留原始题号顺序
