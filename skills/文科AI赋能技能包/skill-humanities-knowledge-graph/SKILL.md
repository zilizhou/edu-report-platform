# 文科课程知识图谱构建技能

## 概述

本技能将文科教学资源（教学大纲、教案、教学PPT、教材章节、文学作品、史料文献、法律条文、哲学原典等）转化为结构化的课程知识图谱。

文科知识的核心特征在于：**概念的历史演变性**、**观点的多元争鸣性**、**文本的阐释依赖性**、**人物与思想的强关联性**。因此，本技能在通用图谱构建方法的基础上，专门设计了适合人文社科知识结构的关系类型体系和抽取规则。

核心产出是一份以**三元组**（主体, 关系, 客体）形式存储的 CSV 文件，可直接导入 Neo4j、Gephi 等图数据库或可视化工具。

## 工作流程

### 第一步：读取教学资源

根据文件类型选择合适的读取方式：

| 文件类型 | 处理方式 |
|---------|---------|
| `.pdf` | 参考 `/mnt/skills/public/pdf-reading/SKILL.md` |
| `.pptx` | 参考 `/mnt/skills/public/pptx/SKILL.md` |
| `.docx` | 参考 `/mnt/skills/public/docx/SKILL.md` |
| `.xlsx` | 参考 `/mnt/skills/public/xlsx/SKILL.md` |
| 纯文本/Markdown | 直接读取内容 |
| 多个文件混合 | 逐一处理后合并知识点池 |

> **提示**：如果用户未上传文件而是粘贴了文本内容，直接从第二步开始处理。

---

### 第二步：知识点抽取

从资源内容中识别并抽取文科知识点。文科知识点可以是：

- **核心概念**：学科基本术语与理论范畴（如"意识形态"、"互文性"、"自然法"）
- **人物**：思想家、作家、历史人物、学派代表（如"卢梭"、"鲁迅"、"亚里士多德"）
- **文本/作品**：经典著作、文学作品、重要文献（如《社会契约论》、《红楼梦》、《查士丁尼法典》）
- **历史事件/时期**：重大事件、历史阶段、运动（如"文艺复兴"、"五四运动"、"启蒙运动"）
- **流派/思潮**：学术流派、文艺思潮、政治运动（如"结构主义"、"浪漫主义"、"法兰克福学派"）
- **理论/学说**：系统性的解释框架（如"历史唯物主义"、"精神分析理论"、"契约论"）
- **制度/体制**：社会组织形式与规则体系（如"三权分立"、"科举制度"、"封建制度"）
- **地域/空间**：与知识密切相关的地理单元（如"古希腊城邦"、"江南文化圈"）
- **方法论**：人文社科研究方法（如"田野调查"、"文本细读"、"比较历史分析"）

**抽取原则：**
1. 优先抽取在资源中**反复出现**、**被重点阐释**或**作为论述起点**的概念
2. 保留概念的**历史语境标记**（如区分"古典自由主义"与"现代自由主义"）
3. 人名、作品名、专有名词保持**规范化全称**（如不省略朝代、国籍等限定信息）
4. 剔除过于泛化的词语（如"社会"、"文化"、"思想"单独出现时）
5. 每个知识点用简洁的名词短语表达（建议 2-15 个字）
6. **观点性知识**也应纳入抽取范围（如"天赋人权说"、"性善论"），不限于中性概念

---

### 第三步：关系类型定义

文科知识图谱采用以下标准化关系类型，覆盖人文社科知识结构的核心维度：

#### 3.1 分类与构成关系

| 关系类型（中文） | 关系标签（英文） | 说明与示例 |
|--------------|--------------|----------|
| 是一种 | `IS_A` | 上下位/分类关系。例：浪漫主义 → IS_A → 文艺思潮 |
| 包含 | `CONTAINS` | 整体-部分关系。例：《诗经》 → CONTAINS → 风雅颂 |
| 属于章节 | `BELONGS_TO` | 知识点归属的章节/模块。例：社会契约论 → BELONGS_TO → 第四章：政治哲学 |
| 实例为 | `HAS_EXAMPLE` | 概念与具体实例。例：意识流小说 → HAS_EXAMPLE → 《尤利西斯》 |

#### 3.2 时间与演变关系

| 关系类型（中文） | 关系标签（英文） | 说明与示例 |
|--------------|--------------|----------|
| 先于 | `PRECEDES` | 时间顺序关系。例：文艺复兴 → PRECEDES → 启蒙运动 |
| 演变为 | `EVOLVES_INTO` | 概念/思潮的历史演变。例：古典经济学 → EVOLVES_INTO → 新古典经济学 |
| 兴起于 | `EMERGED_IN` | 思想/运动的起源时期或地域。例：人文主义 → EMERGED_IN → 14世纪意大利 |
| 影响了 | `INFLUENCED` | 思想、人物或事件的传承影响。例：康德 → INFLUENCED → 黑格尔 |
| 回应了 | `RESPONDS_TO` | 作为对前者的批判、继承或修正。例：实证主义 → RESPONDS_TO → 形而上学 |

#### 3.3 人物与作品关系

| 关系类型（中文） | 关系标签（英文） | 说明与示例 |
|--------------|--------------|----------|
| 由……提出/创作 | `CREATED_BY` | 理论/作品与创作者。例：《君主论》 → CREATED_BY → 马基雅维利 |
| 属于……流派 | `BELONGS_TO_SCHOOL` | 人物/作品与学派归属。例：杜甫 → BELONGS_TO_SCHOOL → 现实主义诗歌 |
| 代表了 | `REPRESENTS` | 人物/作品代表某一思潮或立场。例：伏尔泰 → REPRESENTS → 法国启蒙思想 |
| 批判了 | `CRITIQUES` | 显性批判关系。例：马克思 → CRITIQUES → 黑格尔唯心主义 |
| 继承了 | `INHERITS_FROM` | 思想的正向传承。例：孟子 → INHERITS_FROM → 孔子仁学 |
| 论述于 | `DISCUSSED_IN` | 概念在特定文本中被阐发。例：异化劳动 → DISCUSSED_IN → 《1844年经济学哲学手稿》 |

#### 3.4 逻辑与应用关系

| 关系类型（中文） | 关系标签（英文） | 说明与示例 |
|--------------|--------------|----------|
| 前置知识 | `PREREQUISITE_OF` | 理解后者需先掌握前者。例：古希腊城邦制度 → PREREQUISITE_OF → 柏拉图政治哲学 |
| 用于解释 | `USED_TO_EXPLAIN` | 概念/理论用于分析特定现象。例：阶级分析法 → USED_TO_EXPLAIN → 农民起义 |
| 对比于 | `CONTRASTS_WITH` | 对立/对比关系。例：性善论 → CONTRASTS_WITH → 性恶论 |
| 相关于 | `RELATED_TO` | 横向关联，无明确方向。例：民族主义 ↔ RELATED_TO ↔ 国家主权 |
| 导致 | `LEADS_TO` | 因果或逻辑推导关系。例：重商主义政策 → LEADS_TO → 殖民扩张 |
| 奠基于 | `GROUNDED_IN` | 理论的哲学/方法论基础。例：历史唯物主义 → GROUNDED_IN → 辩证法 |

#### 3.5 争鸣与多元关系（文科特有）

| 关系类型（中文） | 关系标签（英文） | 说明与示例 |
|--------------|--------------|----------|
| 争论于 | `DEBATES_WITH` | 不同学者/流派围绕同一问题存在学术争鸣。例：义利之辨 → DEBATES_WITH → 功利主义 |
| 重新诠释了 | `REINTERPRETS` | 后人对经典文本或概念的新解读。例：新儒家 → REINTERPRETS → 先秦儒学 |
| 本土化为 | `LOCALIZED_AS` | 外来理论在特定文化语境中的转化。例：马克思主义 → LOCALIZED_AS → 毛泽东思想 |
| 隐喻为 | `METAPHORIZED_AS` | 文学/修辞层面的象征或隐喻关系（适用于文学课程）。例：荆棘鸟 → METAPHORIZED_AS → 殉道与追求 |

> **扩展**：如原始资源中存在以上未覆盖的关系，可自定义新关系标签，格式统一为大写英文下划线（如 `TRANSLATED_BY`、`SET_IN`、`AMENDED_BY`）。

---

### 第四步：构建三元组

将所有关系整理为三元组格式：

```
(主体知识点, 关系类型, 客体知识点)
```

**三元组构建规则：**
1. 每条三元组一行，主体和客体均为已抽取的知识点
2. 关系方向要明确（有向图），`RELATED_TO` 和 `DEBATES_WITH` 例外（无向关系，只记录一次）
3. 同一对知识点可以有多种关系（生成多条三元组）
4. 避免冗余：同一关系方向只记录一次
5. 知识点名称在全表中保持一致（区分大小写）
6. **争鸣性观点**的三元组应在 `note` 字段标注"存在学术争议"

## 多文件处理策略

当用户提供多种类型的教学资源时：

1. **分类读取**：按文件类型分批处理
2. **去重合并**：跨文件合并同一知识点（名称规范化后去重，注意区分同名异义概念，如"自由"在不同哲学传统中含义不同，需加限定词区分）
3. **标注来源**：`source` 字段记录每条三元组的来源文件
4. **权重叠加**：同一三元组在多个文件中出现时，`confidence` 提升为 `high`
5. **争议识别**：当同一关系在不同来源中呈现矛盾表述时，标注 `confidence=low` 并在 `note` 字段说明分歧

## 注意事项

### 文科特有处理原则

- **同名异义识别**：文科中同一词汇在不同语境下含义差异大（如"革命"在政治学与文学批评中指向不同），需加限定词区分，如"政治革命"与"文学革命"
- **观点归属标注**：当抽取的是某学者的特定观点而非公认事实时，应通过 `CREATED_BY` 或 `REPRESENTS` 关系将其锚定到具体人物，避免将一家之言呈现为普遍结论
- **时代语境保留**：历史类知识点应尽量在 `time_period` 和 `region` 字段保留时代信息，避免时代错位（anachronism）
- **争鸣关系的处理**：文科中广泛存在学术争鸣，对有争议的三元组不应简单删除，而应标注 `controversy=high` 并在 `note` 字段说明争议焦点
- **文本与观点的区分**：要区分"作品中呈现的内容"与"作者本人的主张"，前者用 `DISCUSSED_IN`，后者用 `CREATED_BY` 或 `REPRESENTS`

### 粒度控制建议

| 课程类型 | 建议知识点数量 |
|---------|-------------|
| 单章节内容 | 20-60 个 |
| 单门课程完整大纲 | 80-300 个 |
| 跨课程综合梳理 | 300-600 个 |

过少说明抽取不够深入；过多说明粒度过细，建议合并细碎概念或按章节分批输出。

### 大文件处理

如文件超过 80 页，建议分章节逐步处理，最后合并。合并时注意跨章节知识点的名称一致性，并补充建立跨章节的 `RELATED_TO` 和 `PREREQUISITE_OF` 关系。
