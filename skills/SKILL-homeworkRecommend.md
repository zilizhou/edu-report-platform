# 个性化作业布置技能

## 概述

本技能综合**学生知识画像**与**课程知识图谱**，考虑知识点间的关联与依赖关系，为每位学生自动生成一份满分 100 分的个性化作业方案。

**有题库**：从题库中智能选题  
**无题库**：根据学生薄弱点现场生成题目  
**两种情况**均可输出完整、可直接使用的作业内容。

产出结果可用于：差异化课后练习、查漏补缺辅导、课前预习清单、自适应练习系统等场景。

---

## 前置依赖

开始前确认以下输入已就绪：

| 输入 | 来源 | 是否必须 |
|-----|------|---------|
| 学生知识画像 | `student-knowledge-profiling` 技能产出，或用户手动提供 | ✅ 必须 |
| 课程知识图谱 | 用户提供（含知识点列表、层级关系、依赖关系） | ✅ 必须 |
| 题库文件 | 用户上传（xlsx / csv / docx / pdf / 文字描述均可） | ❌ 可选 |
| 作业要求说明 | 用户说明（难度偏好、题型偏好、是否分层等） | ❌ 可选 |

> 若用户尚未完成知识画像，引导其先使用 `student-knowledge-profiling` 技能，再回到本技能继续。

## 核心设计原则

### 原则一：知识关联分析优先

学生某知识点薄弱，**根因不一定在该知识点本身**，很可能是其前置知识点或强关联知识点掌握不足。布置作业前必须：

1. 查找该薄弱知识点在知识图谱中的**前置依赖节点**
2. 检查学生在这些前置节点上的掌握度
3. 若前置节点也薄弱，**优先补充前置知识点练习**，再布置目标知识点练习
4. 若前置节点掌握良好，则直接布置目标知识点的强化练习

**示例：**
```
学生 A：分数除法（poor）← 检查 → 分数乘法（fair）← 检查 → 分数的意义（good）
根因定位：分数乘法是瓶颈，作业应先补分数乘法，再练分数除法
```

### 原则二：区分巩固与补漏

- **excellent / good**：不布置该知识点的题目（避免重复刷题，节省学生时间）
- **fair**：布置少量巩固题，以提升稳定性为主
- **weak**：布置适量强化题，并追溯前置知识点
- **poor**：布置较多补基础题 + 前置知识点补习题，难度从低到高递进

### 原则三：满分 100 分弹性分配

每位学生的作业总分固定为 100 分，根据其薄弱知识点数量和严重程度**动态分配各知识点的题目分值**，确保：
- 薄弱/欠缺知识点获得更多分值（练习量更多）
- 一般知识点获得少量分值（巩固为主）
- 优良知识点不分配分值

### 原则四：无题库时生成原创题目

若用户没有题库，本技能根据知识点内容和掌握等级，直接生成题目，要求：
- 题目类型多样（选择、填空、判断、简答、计算、应用题等）
- 难度与掌握等级匹配（poor → 基础题；weak → 中等题；fair → 提升题）
- 题目叙述清晰，包含完整答案和解析

## 工作流程

### 第一步：解析输入数据

**1.1 读取学生知识画像**

优先从 `student_knowledge_profile.csv` 或 `student_summary_profile.csv` 中提取：

| 字段 | 用途 |
|-----|------|
| `student_id` / `student_name` | 学生标识 |
| `knowledge_point` | 知识点名称 |
| `mastery_level` | 掌握等级（excellent/good/fair/weak/poor）|
| `score_rate` | 得分率（用于精确分配题目权重）|

若用户提供的是非标准格式（如手写表格、描述文字），自行解析并建立学生-知识点掌握度映射表。

**1.2 读取课程知识图谱**

提取以下信息构建知识网络：

```
知识点节点：名称、所属章节、难度等级
依赖关系：A → B 表示"学习 B 需要先掌握 A"
关联关系：A ↔ B 表示"A 和 B 密切相关，互相影响"
```

若用户提供的知识图谱格式简单（如无向列表、章节大纲），根据学科常识推断依赖关系，并告知用户。

**1.3 读取题库（如有）**

提取每道题的：题目编号、题目内容、所属知识点、题型、难度、分值、答案、解析。

---

### 第二步：知识点关联分析

对每位学生的每个薄弱/欠缺知识点，执行以下分析：

```
FOR 每个 mastery_level IN [weak, poor]:
    找出该知识点在知识图谱中的所有前置节点
    FOR 每个前置节点:
        查询学生在该前置节点上的掌握度
        IF 前置节点也是 weak 或 poor:
            标记为"根因节点"，加入作业范围，且优先级高于目标知识点
        ELSE:
            该前置节点掌握正常，无需补习
    
    找出与该知识点强关联的横向节点
    IF 强关联节点也薄弱:
        同步加入作业范围，备注"关联补强"
```

输出结果：每位学生的**作业知识点清单**，包含：
- 目标知识点（原本薄弱的）
- 前置补充知识点（根因节点）
- 关联补强知识点
- fair 等级的少量巩固知识点

---

### 第三步：分值规划（满分 100 分）

根据知识点清单，按以下规则分配 100 分：

**分值权重基准：**

| 知识点等级 | 分值权重 | 说明 |
|-----------|---------|------|
| poor（欠缺）| 高权重（基础 20 分/点） | 需大量基础练习 |
| weak（薄弱）| 中高权重（基础 15 分/点）| 需强化练习 |
| poor/weak 的前置根因节点 | 额外 +5～10 分 | 补根因优先 |
| fair（一般，巩固）| 低权重（基础 8 分/点）| 少量巩固 |
| 关联补强节点 | 低权重（基础 5 分/点）| 辅助理解 |

**归一化处理：**

将所有知识点的原始分值加总后，等比缩放至总分 100 分，每个知识点取整（最终汇总若有 1～2 分误差，从权重最低的知识点调整）。

**分值下限保障：**

- 每个纳入作业范围的知识点，分值不低于 5 分（保证至少 1 道有效题目）

---

### 第四步：选题或生成题目

#### 情况 A：有题库

**选题规则：**

1. 按知识点从题库中筛选候选题目
2. 按难度匹配掌握等级：
   - `poor` → 选简单题为主（难度 ≤ 3/5）
   - `weak` → 选中等题为主（难度 3/5）
   - `fair` → 选中等偏难题（难度 3～4/5）
3. 控制题目数量：每个知识点 1～4 道题，使该知识点的题目总分等于规划分值
4. 避免选取该学生在原始考试中已做过的同一题目（若有记录）
5. 若题库中某知识点题目不足，补充生成原创题目（见情况 B）

#### 情况 B：无题库（全部生成）

根据以下要求**现场生成题目**：

**题型配比建议（可根据用户偏好调整）：**

| 分值段 | 题型 |
|-------|------|
| 5～15 分 | 选择题（2分/题）或填空题（3分/题）|
| 15～30 分 | 可加入判断题（2分/题）和简答题（5分/题）|
| 30 分以上 | 可加入应用题/计算题（10分/题）|

**题目生成要求：**

- 题目叙述清晰无歧义，语言符合学段表达习惯
- 每道题必须附有**参考答案**
- 较复杂的题目附**解题思路/解析**（简答题及以上必须）
- 难度阶梯：同一知识点先出基础题，再出应用题
- 避免出现与其他学生重复度极高的题目（可在同一知识点内微调数据或情境）

### 第六步：生成汇总输出文件

#### 文件一：个人作业单（每生一份）

文件名格式：`homework_[student_id]_[student_name].md`（或 `.docx`，按用户偏好）

包含完整作业内容 + 答案解析，如需打印版可参考 `docx` 技能生成 Word 文件。

#### 文件二：`homework_summary.csv`（班级作业汇总表）

每行 = 一位学生的作业规划摘要，便于教师查看全班分配情况。

```csv
student_id,student_name,target_kps,root_cause_kps,reinforcement_kps,total_questions,total_score,difficulty_profile,has_custom_questions
S001,张三,分数除法;分数乘法,分数乘法,分数的意义,8,100,基础为主,是
S002,李四,分数除法,无,分数乘法,5,100,中等偏难,否
S003,王五,分数的意义;小数加减,无,无,4,100,巩固为主,否
```

**字段说明：**

| 字段 | 说明 |
|-----|------|
| `target_kps` | 本次作业主要练习的薄弱知识点 |
| `root_cause_kps` | 根因补充知识点（前置依赖不足） |
| `reinforcement_kps` | fair 等级的巩固知识点 |
| `total_questions` | 作业总题数 |
| `total_score` | 作业总分（固定 100 分）|
| `difficulty_profile` | 难度分布描述 |
| `has_custom_questions` | 是否含现场生成的原创题目 |

#### 文件三：`homework_kp_coverage.csv`（知识点覆盖分析表）

帮助教师了解哪些知识点在全班被重点针对练习。

```csv
knowledge_point,student_count,as_target_count,as_root_cause_count,as_reinforcement_count,total_questions_generated
分数乘法,12,8,4,0,45
分数除法,9,9,0,0,38
分数的意义,15,3,6,6,22
```

### 第七步：质量自检

输出前逐项确认：

- [ ] 每位学生的作业总分恰好为 100 分（允许 ±1 分四舍五入误差）
- [ ] 作业说明中已列出知识点选择的理由（透明化，学生能看懂）
- [ ] 前置知识点分析已执行，根因节点已被优先补充
- [ ] excellent / good 等级知识点未被纳入作业（不重复刷已掌握内容）
- [ ] 题目难度与掌握等级匹配（poor 不出难题，fair 不出基础题）
- [ ] 每道题均有参考答案
- [ ] 若无题库，生成的题目叙述清晰，无歧义
- [ ] `homework_summary.csv` 中所有学生均已覆盖

## 特殊情况处理

| 情况 | 处理方式 |
|-----|---------|
| 学生所有知识点均为 excellent/good | 生成一份拓展提高型作业：选取课程知识图谱中的高阶知识点或综合应用题，满分 100 分 |
| 学生薄弱知识点过多（导致每点分值极低）| 筛选掌握度最低的前 5 个知识点重点攻破，其余知识点在备注中建议课后补习 |
| 学生知识画像数据质量为 low（仅有总分）| 按总得分率估算整体掌握情况，覆盖当前章节所有核心知识点，作业说明中注明"基于整体估算，覆盖全章节" |
| 课程知识图谱未提供依赖关系 | 根据学科常识推断典型前置依赖，列出推断依据告知用户确认 |
| 同一知识点前置链过长（A→B→C→D 均薄弱）| 只追溯 2 层前置，超出部分在作业说明中提示"建议配合教师辅导，从基础重学 [最底层知识点]" |
| 题库中某知识点完全无题 | 现场生成 1～3 道该知识点题目，`has_custom_questions` 标注"是" |
| 班级人数超过 40 人 | 先生成全班汇总表和分析，再按用户需求逐组或逐人生成详细作业单 |

---

## 输出规范

**最终向用户交付：**

1. **每位学生的个性化作业单**（必须）：含题目、答案、解析，格式清晰可打印
2. `homework_summary.csv` — 班级作业汇总表（必须）
3. `homework_kp_coverage.csv` — 知识点覆盖分析表（推荐）
4. **对话中同步输出班级作业设计说明**，包含：
   - 全班作业设计思路概述
   - 最常被纳入作业的 Top 3 薄弱知识点（说明全班共性问题）
   - 发现的前置根因链（如"多位同学分数除法弱，根因是分数乘法不牢"）
   - 是否使用了原创题目及质量说明
   - 若部分学生知识画像数据质量较低，说明对应局限性

## 典型用例示例

**用例 1：有题库 + 小题分知识画像**

用户输入："这是全班知识画像 CSV 和题库，请帮我为每位学生布置个性化作业"

处理步骤：
1. 读取知识画像，提取每位学生的掌握等级
2. 分析知识图谱中的前置依赖，识别根因节点
3. 按分值规划分配 100 分到各知识点
4. 从题库中按知识点 + 难度筛选题目
5. 组装每人作业单，生成汇总 CSV

**用例 2：无题库 + 仅有总分画像**

用户输入："我没有题库，成绩单只有总分，但我想给学生出点个性化练习"

处理步骤：
1. 识别数据质量为 low，告知用户估算局限性
2. 以整体得分率区段推断整体掌握层次
3. 覆盖当前章节全部核心知识点，按难度分层生成原创题目
4. 输出作业单，备注"建议下次收集小题得分以获得更精准的个性化作业"

**用例 3：高分学生全部掌握良好**

用户输入："有几个学生各知识点都是 excellent，我该给他们出什么？"

处理步骤：
1. 识别无薄弱/欠缺知识点
2. 生成拓展型作业：选取课程知识图谱中标注为"高阶"或"综合应用"的知识点
3. 题目以开放性应用题、跨知识点综合题为主
4. 满分同样为 100 分，难度整体上移

---

## 注意事项

- **作业量适度**：单份作业题目数建议控制在 5～15 题，避免学生负担过重；若薄弱点过多，优先攻破最关键的 3～5 个
- **前置分析必须执行**：不可跳过知识关联分析直接出题，根因识别是本技能的核心价值
- **保持学生可读性**：作业说明部分语言应简洁友好，让学生明白"为什么要做这些题"
- **答案完整性**：每道题必须有答案；简答、计算、应用题必须有解析，不得省略
- **隐私保护**：作业文件中含有学生姓名，提醒用户注意文件分发范围，避免学生之间相互看到各自的知识盲点
- **迭代改进**：作业完成后的得分数据可再次输入知识画像技能，形成持续迭代的学情追踪闭环
