# 学生知识画像构建技能

## 概述

本技能综合题目知识点标注与学生成绩数据，为每位学生构建**知识点维度的学习画像**，量化各知识点的掌握程度，识别优势与薄弱点，输出结构化 CSV 文件。

核心设计原则：**在任意精度的成绩数据下，都要体现学生个体差异**。对于粗粒度成绩（仅总分或大题分），通过引入试卷元数据中的难度等级和认知层次信息，对知识点掌握度进行个性化修正，使同分学生之间产生有依据的画像差异，而非千人一面的均等估算。每条画像记录均附置信区间与风险指数，帮助教师准确判断干预优先级。

产出结果可用于：个性化辅导推荐、分层教学设计、学业预警、班级整体学情报告、自适应练习系统等场景。

---

## 前置依赖

在开始前需确认以下两类输入数据均已就绪：

**① 题目知识点标注数据**（来自 `exam-question-tagging` 技能，或用户手动提供）

必须包含以下字段：

| 字段 | 说明 | 是否必须 |
|------|------|---------|
| `question_id` | 题目编号 | ✅ |
| `primary_kp` | 主要知识点 | ✅ |
| `secondary_kp` | 次要知识点（可多个） | ✅ |
| `score` | 题目满分 | ✅ |
| `difficulty` | 难度等级：`easy` / `medium` / `hard` | ✅ |
| `cognitive_level` | 认知层次：`remember` / `understand` / `apply` / `analyze` | ✅ |
| `question_type` | 题型（如选择题、简答题） | ❌ |

> `difficulty` 和 `cognitive_level` 是个性化画像的核心依据。若缺失，需提示用户补充；若确实无法补充，则跳过个性化调整步骤，在 `note` 字段注明，并将 `data_quality` 降一级。

**② 学生成绩数据**（用户上传的成绩单文件）

成绩单格式分三类，详见第二步。

> 若用户尚未完成题目知识点标注，应先引导其使用 `exam-question-tagging` 技能处理试卷，再回到本技能继续。

## 工作流程

### 第一步：读取并预处理输入文件

根据文件类型选择对应读取方式：

| 文件类型 | 处理方式 |
|---------|---------|
| `.xlsx` / `.csv` | 参考 `/mnt/skills/public/xlsx/SKILL.md` |
| `.pdf` | 参考 `/mnt/skills/public/pdf-reading/SKILL.md` |
| `.docx` | 参考 `/mnt/skills/public/docx/SKILL.md` |
| 纯文本/Markdown | 直接读取，按行或表格结构解析 |

读取后提取并确认以下信息：

- **学生基本信息**：学生总数、学生标识字段（姓名/学号/两者皆有）
- **考试信息**：试卷/作业名称、试卷满分
- **成绩单格式类型**：见第二步
- **班级基准统计**：计算全班总分均值与标准差、各大题的班级平均得分率（供后续个性化调整使用）
- **知识点元数据预处理**：
  - 为每个知识点计算加权平均难度权重（以关联题目满分为权重）
  - 为每个知识点计算认知层次构成（各层次关联题目满分占比）
  - 统计每个知识点的关联题目数量和总满分

---

### 第二步：识别成绩单格式并计算知识点掌握度

成绩单分三种格式，对应不同的计算路径。每种格式均包含基础计算和个性化调整两个阶段。

---

#### 类型 A：仅含总分

**特征**：成绩单只有学生标识和一个总分列，无题目或大题分数。

```
示例：
学号,姓名,总分
2021001,张三,85
2021002,李四,72
```

**计算分四步：**

**步骤 A1：基础得分率**

```
基础得分率(student, kp) = 学生总分 ÷ 试卷满分
```

此时所有知识点的基础得分率相同，均等于总得分率。

**步骤 A2：难度感知调整**

核心逻辑：相同总分下，高难度知识点的得分差异更能反映学生真实能力。总分高于班级均值的学生，其高难度知识点掌握度应上调；总分低于均值的学生，其低难度知识点掌握度应下调——因为连简单题都丢分，说明基础漏洞更严重。

```
班级总分均值率 = 全班总分均值 ÷ 试卷满分
学生相对偏差(student) = 学生总分率 - 班级总分均值率

知识点难度权重(kp)：
  该kp关联题目全为 easy              → 难度权重 = 0.10
  全为 medium 或 easy/medium 混合    → 难度权重 = 0.25
  含任意 hard 题目                   → 难度权重 = 0.50

难度调整系数(student, kp) = 1 + 学生相对偏差(student) × 难度权重(kp)

难度调整后得分率(student, kp) = 基础得分率(student) × 难度调整系数(student, kp)
                               结果截断至 [0.00, 1.00]
```

**步骤 A3：认知层次修正**

核心逻辑：「记忆」类题目班级整体差异小（背了就会），「应用/分析」类题目差异大（举一反三能力分化）。高分学生在应用层次的优势更突出，低分学生在记忆层次的劣势相对较小。

```
认知层次偏移系数：
  remember   → +0.05（高/低分学生差距小，向均值收缩）
  understand → 0.00（中性，不调整）
  apply      → -0.05（高/低分差距大，进一步拉开）
  analyze    → -0.10

知识点认知层次权重(kp) =
  Σ(该kp各关联题目的认知层次偏移系数 × 题目满分) ÷ Σ(题目满分)

最终得分率(student, kp) =
  难度调整后得分率(student, kp) + 学生相对偏差(student) × 知识点认知层次权重(kp)
  结果截断至 [0.00, 1.00]
```

**步骤 A4：置信区间**

```
置信区间宽度(kp) = max(0.10, 0.20 - 该kp关联题目数 × 0.02)
上限为 ±0.20，最窄为 ±0.10（关联题目越多，区间越窄）
```

---

#### 类型 B：含大题分 + 总分

**特征**：成绩单包含各大题（题组）的得分及总分，但无具体小题分数。

```
示例：
姓名,选择题(20分),填空题(20分),简答题(30分),综合题(30分),总分
张三,16,14,22,23,75
李四,12,18,20,20,70
```

**基础计算：**

```
大题得分率(student, section) = 大题实际得分 ÷ 大题满分

知识点掌握度(student, kp) =
  Σ(大题得分率 × 该大题中该kp关联题目加权满分) ÷ Σ(该kp加权满分)

加权满分说明：primary_kp 权重 0.7，每个 secondary_kp 权重 = 0.3 ÷ secondary_kp数量
```

**大题内部难度分层调整：**

类型B已能区分不同大题，但同一大题内各知识点得分率仍相同。利用大题得分率与知识点难度，在大题内部做二次区分：

```
大题难度基准(section) = 该大题所有题目的班级平均得分率

大题内难度调整系数(student, kp, section) =
  1 + (大题得分率(student, section) - 大题难度基准(section)) × 知识点难度权重(kp)

最终知识点得分率(student, kp) =
  Σ(大题得分率 × 大题内难度调整系数 × kp在该大题的加权满分) ÷ Σ(kp加权满分)
  结果截断至 [0.00, 1.00]
```

**认知层次修正**（同类型A步骤A3，以大题得分率偏差替代总分率偏差作为基准）：

```
大题认知修正后得分率(student, kp) =
  最终知识点得分率(student, kp)
  + (大题得分率(student, section) - 大题难度基准(section)) × 知识点认知层次权重(kp)
  结果截断至 [0.00, 1.00]
```

**置信区间：**

```
置信区间宽度(kp) = max(0.05, 0.10 - 该kp跨越的大题数 × 0.02)
上限为 ±0.10，最窄为 ±0.05（跨越大题越多，区间越窄）
```

---

#### 类型 C：含小题分 + 总分

**特征**：成绩单包含每道小题的得分及总分，精度最高。

```
示例：
姓名,Q1(2分),Q2(2分),Q3(3分),简答1(6分),综合1(10分),总分
张三,2,1,3,5,8,19
李四,1,2,2,4,9,18
```

**精确计算：**

```
知识点掌握度(student, kp) =
  Σ(各关联题目实际得分 × 该题对kp的权重) ÷ Σ(各关联题目满分 × 权重)

权重分配：primary_kp = 0.7；每个 secondary_kp = 0.3 ÷ secondary_kp数量
```

**答题一致性检验**（当同一知识点关联多道题时）：

```
一致性标准差 = 该kp各关联题目实际得分率的标准差

标准差 > 0.30 → note 注明「该知识点得分不稳定，可能存在偶然因素」
标准差 ≤ 0.10 → note 注明「得分稳定，掌握度可信」
```

**置信区间：**

```
置信区间宽度(kp) = max(0.03, 该kp关联题目实际得分率的标准差 × 0.5)
```

### 第三步：知识点掌握度等级映射

综合第二步计算结果，为每位学生每个知识点映射掌握等级。

**掌握等级映射（三种类型统一）：**

| 得分率区间 | 掌握等级 | 标签 | 含义 |
|-----------|---------|------|------|
| ≥ 90% | 优秀 | `excellent` | 完全掌握，可迁移应用 |
| 75%～89% | 良好 | `good` | 基本掌握，偶有失误 |
| 60%～74% | 一般 | `fair` | 部分掌握，存在漏洞 |
| 40%～59% | 薄弱 | `weak` | 掌握不足，需重点练习 |
| < 40% | 欠缺 | `poor` | 严重欠缺，需系统补习 |

> **置信区间跨等级处理**：当置信区间横跨两个等级边界时，保留主等级，在 `note` 字段注明「实际等级可能为X或Y，建议结合其他数据综合判断」。

**数据可信度标注（`data_quality`）：**

| 成绩单类型 | 数据质量 | 说明 |
|-----------|---------|------|
| 类型C（小题分）| `high` | 精确到每道题 |
| 类型B（大题分）含大题内难度调整 | `medium-high` | 大题内部进一步分层 |
| 类型B（大题分）基础计算 | `medium` | 精确到题组 |
| 类型A（总分）含难度与认知层次调整 | `medium-low` | 统计推断个性化 |
| 类型A（总分）基础估算（无题目元数据）| `low` | 整体均等估算 |

---

### 第四步：个性化扩展维度计算

在知识点掌握度之外，补充两个个性化维度，使同分学生产生多角度的差异化画像。

#### 4.1 认知画像

统计学生在不同认知层次知识点上的综合表现，识别其认知强项与弱项：

```
认知层次得分率(student, cognitive_level) =
  该认知层次所有关联题目的加权平均实际得分率（以题目满分为权重）

认知强项 = 得分率最高的认知层次
认知弱项 = 得分率最低的认知层次
```

输出至 `student_summary_profile.csv` 的 `cognitive_strength` 和 `cognitive_weakness` 字段。

> 对于类型A/B，认知层次得分率由调整后的知识点掌握度加权反推，而非精确计算，在 `note` 中注明「认知画像为估算」。

#### 4.2 知识点风险指数

综合掌握度和置信区间，量化每个知识点对该学生的教学干预优先级：

```
风险指数(student, kp) = (1 - score_rate) × (1 + 置信区间宽度)

风险等级：
  风险指数 > 0.60     → high_risk（优先干预）
  风险指数 0.30～0.60  → medium_risk（需关注）
  风险指数 < 0.30     → low_risk（暂无需特别干预）
```

输出至 `student_knowledge_profile.csv` 的 `risk_index` 字段。

### 第五步：多次考试数据合并（可选）

若用户提供同一学生的多次考试成绩，需合并计算综合画像：

**合并规则：**

1. 以**时间加权平均**为主：默认最近一次权重 0.5，前一次 0.3，更早 0.2；可由用户调整
2. 若用户明确说明「以最终考试为准」，则只使用最后一次数据
3. 不同类型（A/B/C）的成绩可合并，但需按数据质量系数折算权重：

```
合并权重 = 时间权重 × 数据质量系数

数据质量系数：
  high        → 1.00
  medium-high → 0.85
  medium      → 0.70
  medium-low  → 0.50
  low         → 0.30
```

**趋势标签（`trend`）：**

| 标签 | 条件 |
|------|------|
| `improving` | 最近一次比前一次得分率提升 ≥ 10% |
| `stable` | 波动 < 10% |
| `declining` | 最近一次比前一次得分率下降 ≥ 10% |
| `new` | 该知识点仅出现在最近一次考试中，无历史对比 |

若提供三次及以上考试数据，额外生成 `student_trend_profile.csv`，横向展示各次得分率变化：

```csv
student_id,student_name,knowledge_point,exam1_rate,exam2_rate,exam3_rate,trend,weighted_avg_rate
S001,张三,TCP协议,0.60,0.75,0.83,improving,0.75
```

### 第六步：生成学生画像 CSV 文件

每行 = 一个学生在一个知识点上的画像记录。

```csv
student_id,student_name,exam_id,knowledge_point,score_rate,confidence_interval,mastery_level,risk_index,risk_level,data_quality,related_questions,earned_score,total_score,trend,note

```

**字段说明：**

| 字段 | 类型 | 是否必填 | 说明 |
|-----|------|---------|------|

| `student_name` | string | ✅ | 学生姓名 |
| `exam_id` | string | ✅ | 考试/作业标识（如"期末考试"、"第3次作业"）|
| `knowledge_point` | string | ✅ | 知识点名称 |
| `score_rate` | float | ✅ | 掌握度得分率（0.00～1.00，保留两位小数）|
| `confidence_interval` | string | ✅ | 置信区间，如 `±0.15` |
| `mastery_level` | string | ✅ | 掌握等级标签（excellent / good / fair / weak / poor）|
| `risk_index` | float | ✅ | 知识点风险指数（0.00～2.00，保留两位小数）|
| `risk_level` | string | ✅ | 风险等级（high_risk / medium_risk / low_risk）|
| `data_quality` | string | ✅ | 数据可信度（high / medium-high / medium / medium-low / low）|
| `related_questions` | string | ✅ | 关联题目编号，用 `;` 分隔 |
| `earned_score` | number | ❌ | 该知识点实际得分（类型A时留空）|
| `total_score` | number | ❌ | 该知识点满分（类型A时留空）|
| `trend` | string | ❌ | 趋势标签（多次考试时填写）|
| `note` | string | ❌ | 备注（调整说明、置信区间跨等级提示、一致性说明等）|

```csv
knowledge_point,class_avg_score_rate,class_score_std,excellent_count,good_count,fair_count,weak_count,poor_count,total_students,mastery_rate,alert_level,difficulty_level,cognitive_level
```

**字段说明：**

| 字段 | 类型 | 说明 |
|-----|------|------|

| `class_avg_score_rate` | float | 全班平均得分率 |
| `class_score_std` | float | 全班得分率标准差（越大说明两极分化越严重）|
| `excellent_count` … `poor_count` | number | 各等级人数 |
| `total_students` | number | 参与统计的总学生数 |
| `mastery_rate` | float | 达到 good 及以上的学生比例 |
| `alert_level` | string | 预警等级：`normal`（≥60%）/ `warning`（40%～59%）/ `high_alert`（<40%）|
| `difficulty_level` | string | 该知识点的加权平均难度（easy / medium / hard）|
| `cognitive_level` | string | 该知识点的主要认知层次 |

---

### 第七步：质量自检

输出前逐项确认：

- [ ] 所有学生的 `student_id` 唯一，无重复
- [ ] `score_rate` 值在 0.00～1.00 之间，无负数或超过 1 的异常值
- [ ] `confidence_interval` 宽度符合对应格式类型规范：类型A ≤ ±0.20，类型B ≤ ±0.10，类型C 动态计算
- [ ] 类型A/B 的 `note` 字段包含个性化调整说明，不得与精确数据混用而不加区分
- [ ] `knowledge_point` 名称与题目知识点标注文件中的名称完全一致，无擅自改名或合并
- [ ] `top3_strength` 和 `top3_weakness` 无重叠
- [ ] 班级热力图的 `total_students` 与成绩单学生总数一致
- [ ] **个性化验证**（类型A/B必检）：抽取总分相同或相近的两名学生，确认其高难度知识点与低难度知识点的 `score_rate` 已产生差异，不完全相同
- [ ] CSV 编码为 UTF-8，所有数值字段无空格或非数字字符

## 特殊情况处理

| 情况 | 处理方式 |
|-----|---------|
| 缺少 `difficulty` / `cognitive_level` 字段 | 提示用户补充；确实无法补充时跳过个性化调整，`data_quality` 降一级，`note` 注明 |
| 成绩单缺少部分学生某题得分 | 该题记为缺考（`earned_score=null`），不参与计算，`note` 注明「缺考」 |
| 同一学生在成绩单中出现多行 | 以最高分为准，`note` 注明「取最高分」 |
| 置信区间跨越等级边界 | `mastery_level` 保留主等级，`note` 注明「实际等级可能为X或Y」 |
| 试卷满分与成绩单最高分不一致 | 以用户提供的试卷满分为准，若未提供则取成绩单最大值，提示用户确认 |
| 学生总分为满分或 0 分 | 直接设为 1.00 或 0.00，不做难度调整（避免系数放大误差）|
| 班级人数少于 10 人 | `alert_level` 和统计调整结果仅供参考，摘要中说明样本量不足 |
| 所有知识点难度相同 | 跳过难度感知调整，`note` 注明「该试卷知识点难度均一，无法进行难度分层调整」 |
| 某知识点仅关联 1 道题 | 类型C一致性检验跳过；类型A/B置信区间加宽至 ±0.25 |
| 多次考试成绩单类型不同（如A+C混合）| 以数据质量最高的为主导，合并权重 = 时间权重 × 数据质量系数 |
| 成绩单同时含大题分和小题分 | 优先以小题分（类型C）计算，大题总分仅用于校验（二者之和应一致）|

## 注意事项

- **隐私保护**：成绩数据属于学生个人隐私，本技能不持久化任何学生数据。
- **估算透明**：类型A/B的掌握度为统计推断估算值，必须在 `confidence_interval` 字段和对话摘要中明确标注，不得与精确数据混用而不加区分。
- **调整系数截断**：难度调整与认知层次调整叠加后，结果超出 [0.00, 1.00] 时截断处理，不外推。
- **避免过度解读**：单次考试的得分率受题目抽样、难度、临场发挥等多重因素影响，应提醒用户结合多次数据综合判断，不宜据此做出重大教学决定。
- **知识点名称对齐**：本技能的知识点名称须与 `exam-question-tagging` 产出严格一致，不得擅自改名、拆分或合并。
