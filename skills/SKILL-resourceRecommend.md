# 大学生个性化学习资源推荐技能

## 概述

本技能综合**学生知识画像**与**课程知识图谱**，为每位大学生推荐一份个性化的学习资源清单。

**资源来源**：本地资源（教师上传的课件、教材章节、实验指导）+ 网络资源（公开课、论文、工具、代码仓库等）  
**资源形式**：视频、文章/教材、交互平台、代码/项目、播客/音频、可视化工具等多模态  
**核心逻辑**：知识掌握等级 × 知识关联分析 → 精准匹配资源难度与深度  

面向**高校大学生**，强调学术深度、自主探索能力和实践应用，而非中小学式的基础讲解。

---

## 前置依赖

| 输入 | 来源 | 是否必须 |
|-----|------|---------|
| 学生知识画像 | `student-knowledge-profiling` 技能产出，或用户手动提供 | 必须 |
| 课程知识图谱 | 用户提供（含知识点列表、层级/依赖关系、学科方向） | 必须 |
| 本地学习资源库 | 用户上传（课件、教材、实验材料、参考文献等） | 可选 |
| 课程背景说明 | 用户说明（学科、年级、课程目标、语言偏好等） | 可选 |

> 若用户尚未完成知识画像，引导其先使用 `student-knowledge-profiling` 技能，再回到本技能继续。

## 核心设计原则

### 原则一：掌握等级驱动资源深度

不同掌握等级的学生，所需的资源类型和深度有本质差异：

| 掌握等级 | 学习需求定位 | 资源策略 |
|---------|------------|---------|
| poor（欠缺）| 重建认知框架 | 系统性入门课程、直观动画讲解、基础教材章节 |
| weak（薄弱）| 填补理解漏洞 | 专题讲解视频、有例题的笔记/博客、配套练习平台 |
| fair（一般）| 强化与应用 | 进阶课程片段、经典习题集、案例分析文章 |
| good（良好）| 拓展与融会贯通 | 学术综述、开源项目实践、跨知识点综合资源 |
| excellent（优秀）| 前沿探索 | 顶会论文、前沿工具/框架、研究级项目、学术讲座 |

**大学生特别注意**：即使是 poor 等级，推荐资源也应保持大学本科/研究生的学术调性，不推荐中小学风格的基础资源。

### 原则二：知识关联分析优先

薄弱知识点的根因可能是前置知识点不牢。推荐资源前须：

1. 沿知识图谱追溯薄弱点的**前置依赖节点**
2. 若前置节点也薄弱 → 优先推荐前置节点的资源（标注"建议先学"）
3. 若强关联横向节点也薄弱 → 推荐能同时覆盖两者的综合资源

### 原则三：多模态匹配学习场景

大学生学习场景多样，不同形式的资源服务不同场景：

| 资源形式 | 适用场景 | 典型来源 |
|---------|---------|---------|
| 视频课程 | 系统建立知识框架，通勤/睡前学习 | Bilibili、YouTube、中国大学MOOC、Coursera、MIT OCW |
| 文章/博客 | 深度理解某一细节，快速查阅 | 知乎、CSDN、Medium、官方文档、维基百科 |
| 学术论文 | 理解原理来源，前沿探索 | arXiv、知网、Google Scholar、Semantic Scholar |
| 交互平台 | 边学边练，即时反馈 | LeetCode、Kaggle、Khan Academy、Brilliant.org |
| 代码/项目 | 实践应用，工程能力培养 | GitHub、Gitee、Papers with Code |
| 可视化工具 | 理解抽象概念，直觉建立 | Desmos、3Blue1Brown、VisuAlgo、Wolfram Alpha |
| 播客/音频 | 碎片时间，宏观视角 | 各学科学术播客、TED Talks |
| 教材/讲义 | 系统学习，考试备考 | 用户本地上传或公开电子版 |

### 原则四：优先推荐真实可访问的资源

- **优先推荐有明确平台名称的真实资源**，避免虚构不存在的资源链接
- 对于可能需要科学上网的资源（如 YouTube、Coursera），注明访问要求
- 对于付费资源，注明"付费"或"部分免费"
- 若不确定某资源是否仍可访问，使用保守描述（"可在 X 平台搜索关键词 Y"）而非直接给出可能失效的 URL

## 工作流程

### 第一步：解析输入数据

**1.1 读取学生知识画像**

从知识画像中提取每位学生的知识点掌握等级（excellent/good/fair/weak/poor）和得分率。

**1.2 读取课程知识图谱**

提取知识点的依赖关系、章节归属、学科方向（理工/人文/社科/艺术等），用于判断资源适用性和追溯前置依赖链。

**1.3 了解课程背景（如用户未说明，主动询问以下关键信息）**

- 学科/课程名称（如：数据结构、高等数学、宏观经济学）
- 年级与层次（本科低年级/高年级/研究生）
- 语言偏好（中文优先 / 英文可接受 / 均可）
- 是否有校内资源库（图书馆数据库、内网课件）
- 学生每周课外学习可用时长（用于估算推荐资源总量）

**1.4 盘点本地资源（如用户提供）**

提取本地资源的知识点覆盖范围，优先为学生匹配本地资源，再补充网络资源。

---

### 第二步：知识点关联分析

对每位学生执行前置依赖链分析，输出**资源推荐知识点优先级清单**：

- **P0**：前置根因节点（必须先补，标注"建议先学"）
- **P1**：主要薄弱/欠缺知识点（重点推荐）
- **P2**：一般掌握的知识点（少量巩固拓展）
- **P3**：良好/优秀知识点（前沿探索，可选）

```
FOR 每个 mastery_level IN [weak, poor]:
    沿知识图谱找出所有前置节点
    若前置节点也薄弱 → 标记 P0，优先推荐其资源
    若强关联横向节点也薄弱 → 寻找能同时覆盖两者的综合资源
```

---

### 第三步：资源匹配与生成

#### 3.1 本地资源优先匹配

若用户提供本地资源，先按知识点映射分配最相关的本地材料给对应优先级知识点。

#### 3.2 按学科推荐网络资源

**理工科（数学、物理、计算机、工程等）：**

| 知识点等级 | 优先推荐资源类型 | 典型平台 |
|-----------|--------------|---------|
| poor | 系统性视频课程（完整章节）| 中国大学MOOC、Bilibili 高校公开课、MIT OCW |
| weak | 专题讲解 + 练习平台 | Bilibili、LeetCode、牛客网、洛谷 |
| fair | 进阶教程 + 典型例题 | GitHub 学习笔记仓库、Stack Overflow |
| good | 开源项目 + 学科综述 | GitHub、Papers with Code、arXiv |
| excellent | 顶会论文 + 前沿框架 | arXiv、ACM DL、IEEE Xplore |

**人文社科（历史、哲学、经济、管理等）：**

| 知识点等级 | 优先推荐资源类型 | 典型平台 |
|-----------|--------------|---------|
| poor | 权威教材章节 + 概念讲解视频 | 中国大学MOOC、Coursera、知乎专栏 |
| weak | 深度长文 + 案例分析 | 知乎、学术公众号、豆瓣书单 |
| fair | 学术文献导读 + 名家访谈 | 知网、SSRN、TED Talks |
| good | 学术期刊文章 + 读书笔记 | CNKI、JSTOR、Google Scholar |
| excellent | 前沿论文 + 国际会议资料 | SSRN、各学科顶刊官网 |

**艺术设计类：**

| 知识点等级 | 优先推荐资源类型 | 典型平台 |
|-----------|--------------|---------|
| poor | 基础技法视频教程 | Bilibili、Adobe 官方教程、站酷学院 |
| weak | 专项练习教程 + 临摹资源 | Dribbble、Behance、Pinterest |
| fair | 优秀作品赏析 + 设计评论 | ZCOOL、站酷、国外设计博客 |
| good/excellent | 设计竞赛、前沿工具、大师讲座 | TED、国际设计周、Figma 社区 |

#### 3.3 每个知识点推荐资源数量

| 优先级 | 推荐资源数 | 形式要求 |
|-------|-----------|---------|
| P0（根因节点）| 2～3 个 | 至少 1 个视频 + 1 个文本资源 |
| P1（主要薄弱）| 3～5 个 | 多模态（视频 + 文本 + 练习/工具）|
| P2（一般巩固）| 1～2 个 | 以文本/练习为主 |
| P3（前沿拓展）| 1～3 个 | 论文/项目/讲座优先 |

### 第六步：质量自检

输出前逐项确认：

- [ ] 推荐的网络资源为真实存在的平台/资源，未虚构 URL
- [ ] 前置依赖分析已执行，根因节点资源已标注"建议先学"
- [ ] 每位学生的推荐包含多模态形式（至少视频 + 文本两种）
- [ ] 资源难度与掌握等级匹配（poor 不推荐顶会论文，excellent 不推荐纯入门视频）
- [ ] 付费资源、需科学上网的资源均已标注访问方式
- [ ] 学习路径建议具体可执行，非泛泛而谈
- [ ] 本地资源（如有）已优先匹配
- [ ] 资源学术调性符合大学生水平

## 特殊情况处理

| 情况 | 处理方式 |
|-----|---------|
| 所有知识点均 excellent | 推荐前沿资源包：顶会论文、最新工具/框架、学术讲座、国际竞赛，附学术探索路径 |
| 所有知识点均 poor | 推荐系统性全课程资源（MOOC 完整课程），附分阶段学习计划，避免推荐过多零散资源 |
| 中文资源稀缺（冷门领域）| 优先推荐英文资源，标注"建议配合翻译工具使用"，同时推荐中文社区讨论区 |
| 课程为实践型（实验/设计课）| 优先推荐工具教程、开源项目、案例库，而非纯理论讲解 |
| 考试前时间紧迫 | 推荐"最小学习路径"：精选 3～5 个最高价值资源，给出明确时间分配建议 |
| 涉及有版权教材 | 仅推荐合法获取方式（图书馆借阅、正版购买、官方开放版），不推荐盗版资源 |
| 知识画像数据质量为 low | 按整体得分率估算层次，推荐覆盖全章节的系统性资源，并说明精准度限制 |

## 输出规范

**最终向用户交付：**

1. **每位学生的个性化资源推荐单**（必须）：含资源名称、平台、访问方式、内容简介、学习路径建议
2. `resource_recommendation_summary.csv` — 班级推荐汇总（必须）
3. `resource_pool.csv` — 全班资源池（推荐，便于教师审核）
4. **对话中同步输出班级资源推荐说明**：
   - 全班最常推荐的 Top 5 资源（反映共性薄弱点）
   - 前置根因链及对应资源策略
   - 本地 vs 网络资源比例
   - 涉及付费/科学上网资源的汇总（教师可统一处理）
   - 数据质量局限性说明

## 典型用例示例

**用例 1：计算机专业课（有本地课件 + 知识画像）**

步骤：读取画像 → 分析知识图谱依赖（如图论依赖树结构，树结构依赖递归）→ 优先匹配本地课件 → 补充 Bilibili/LeetCode/GitHub 资源 → poor/weak 推荐系统视频课程，good/excellent 推荐开源项目和算法竞赛资源。

**用例 2：经济学课（无本地资源，仅有总分画像）**

步骤：识别数据质量 low → 按整体得分率估算掌握层次 → 低分学生推荐 MOOC + 曼昆教材对应章节，高分学生推荐顶刊论文导读 + IMF/World Bank 报告 → 说明因数据粒度限制，推荐按整体层次而非知识点精准匹配。

**用例 3：研究生课程（所有学生掌握良好）**

步骤：识别大多数知识点 good/excellent → 重点推荐 arXiv 最新论文、Papers with Code、NeurIPS/ICML 讲座、Hugging Face 社区 → 按各学生的细分方向差异（CV/NLP/RL）推荐不同前沿资源 → 附学术工具资源（Weights & Biases、Overleaf）。

## 注意事项

- **资源真实性第一**：宁用"在 X 平台搜索关键词 Y"的保守描述，也不虚构可能失效的 URL
- **大学生学术调性**：即使补基础，也推荐有学术深度的资源，避免过于低幼的教学内容
- **知识关联必须执行**：不可跳过前置依赖分析，根因资源标注"建议先学"是本技能的核心价值
- **版权合规**：仅推荐合法获取方式，不推荐盗版或侵权渠道
- **学习路径具体化**：不要只列资源，要给出"先学什么、再学什么、用多久"的可执行路径
- **隐私保护**：推荐单含学生薄弱点信息，提醒教师注意分发范围
- **动态更新提醒**：网络资源可能随时失效，建议教师每学期审核一次资源池
- **闭环建议**：推荐单使用后，建议收集学生反馈（哪些资源有帮助），持续优化推荐策略
