# 课程知识图谱构建技能

## 概述

本技能将教学资源（教学大纲、教案、教学PPT、电子书籍等）转化为结构化的课程知识图谱。核心产出是一份以**三元组**（主体, 关系, 客体）形式存储的 CSV 文件，可直接导入 Neo4j、Gephi 等图数据库或可视化工具。

---

## 工作流程

### 第一步：读取教学资源

根据文件类型选择合适的读取方式：

| 文件类型 | 处理方式 |
|---------|---------|
| `.pdf` | 参考 `/mnt/skills/public/pdf-reading/SKILL.md` |
| `.pptx` | 参考 `/mnt/skills/public/pptx/SKILL.md` |
| `.docx` | 参考 `/mnt/skills/public/docx/SKILL.md` |
| `.xlsx` | 参考 `/mnt/skills/public/xlsx/SKILL.md` |
| 纯文本/Markdown | 直接读取内容 |
| 多个文件混合 | 逐一处理后合并知识点池 |

> **提示**：如果用户未上传文件而是粘贴了文本内容，直接从第二步开始处理。

---

### 第二步：知识点抽取

从资源内容中识别并抽取知识点。知识点可以是：

- **概念**：术语、定义、原理（如"神经网络"、"梯度下降"）
- **方法/技术**：具体操作步骤或算法（如"反向传播算法"）
- **模型/框架**：理论模型或工具框架（如"LSTM模型"）
- **属性/特征**：描述事物性质的要素（如"激活函数"、"学习率"）
- **过程/流程**：有序的操作序列（如"模型训练流程"）
- **人物/历史**：提出者、发展历程（如"图灵"、"深度学习发展史"）

**抽取原则：**
1. 优先抽取在资源中**反复出现**或**被重点标注**的概念
2. 保持术语的**规范化**（如统一使用全称或公认缩写）
3. 剔除过于笼统的词语（如"方法"、"内容"）
4. 每个知识点用简洁的名词短语表达（建议 2-10 个字）

---

### 第三步：关系类型定义

在知识点之间建立以下标准化关系类型：

| 关系类型（中文） | 关系标签（英文） | 说明 |
|--------------|--------------|------|
| 是一种 | `IS_A` | 上下位/分类关系（如：卷积神经网络 → IS_A → 神经网络）|
| 包含 | `CONTAINS` | 整体-部分关系（如：神经网络 → CONTAINS → 隐藏层）|
| 前置知识 | `PREREQUISITE_OF` | 学习依赖关系（如：线性代数 → PREREQUISITE_OF → 矩阵分解）|
| 用于 | `USED_FOR` | 应用关系（如：正则化 → USED_FOR → 防止过拟合）|
| 相关于 | `RELATED_TO` | 横向关联（如：过拟合 ↔ RELATED_TO ↔ 欠拟合）|
| 对比于 | `CONTRASTS_WITH` | 对立/对比（如：监督学习 → CONTRASTS_WITH → 无监督学习）|
| 导致 | `LEADS_TO` | 因果关系（如：学习率过大 → LEADS_TO → 梯度爆炸）|
| 属于章节 | `BELONGS_TO` | 知识点归属的章节/模块（如：梯度下降 → BELONGS_TO → 第三章：优化算法）|
| 由提出 | `PROPOSED_BY` | 知识与提出者关系（如：反向传播 → PROPOSED_BY → Rumelhart）|
| 实例为 | `HAS_EXAMPLE` | 概念与具体实例（如：激活函数 → HAS_EXAMPLE → ReLU）|

> **扩展**：如原始资源中存在以上未覆盖的关系，可自定义新关系标签，格式统一为大写英文下划线（如 `EXTENDS`、`COMPARED_WITH`）。

---

### 第四步：构建三元组

将所有关系整理为三元组格式：

```
(主体知识点, 关系类型, 客体知识点)
```

**三元组构建规则：**
1. 每条三元组一行，主体和客体均为已抽取的知识点
2. 关系方向要明确（有向图）
3. 同一对知识点可以有多种关系（生成多条三元组）
4. 避免冗余：同一关系方向只记录一次
5. 知识点名称在全表中保持一致（区分大小写）

---

### 第五步：生成 CSV 文件

#### 主文件：`knowledge_graph_triples.csv`

```csv
subject,relation,object,subject_type,object_type,source,confidence
神经网络,IS_A,机器学习模型,概念,概念,第二章PPT,high
卷积神经网络,IS_A,神经网络,概念,概念,教学大纲,high
神经网络,CONTAINS,隐藏层,概念,概念,第二章PPT,high
反向传播,PREREQUISITE_OF,神经网络训练,方法,过程,教案,medium
```

**字段说明：**

| 字段 | 类型 | 说明 |
|-----|------|------|
| `subject` | string | 三元组主体（知识点名称）|
| `relation` | string | 关系类型标签（英文大写）|
| `object` | string | 三元组客体（知识点名称）|
| `subject_type` | string | 主体知识点类型（概念/方法/模型/过程/属性/人物）|
| `object_type` | string | 客体知识点类型（同上）|
| `source` | string | 知识点来源（如"第三章教案"、"教学大纲"）|
| `confidence` | string | 置信度（high/medium/low）|

#### 辅助文件：`knowledge_nodes.csv`（知识点节点表）

```csv
node_id,name,type,chapter,definition,difficulty
KP001,神经网络,概念,第二章,由大量神经元连接构成的计算模型,medium
KP002,卷积神经网络,模型,第四章,专为图像处理设计的神经网络结构,hard
```

**字段说明：**

| 字段 | 类型 | 说明 |
|-----|------|------|
| `node_id` | string | 唯一标识符（格式：KP + 三位数字）|
| `name` | string | 知识点名称 |
| `type` | string | 知识点类型 |
| `chapter` | string | 所属章节 |
| `definition` | string | 简要定义（来自原资源，可为空）|
| `difficulty` | string | 难度等级（easy/medium/hard）|

---

### 第六步：质量检验

输出前进行以下自检：

- [ ] 所有三元组的 subject 和 object 均出现在节点表中
- [ ] 无孤立节点（每个知识点至少有一条关系）
- [ ] 关系标签拼写正确，格式统一
- [ ] CSV 无空行、无多余逗号、编码为 UTF-8
- [ ] 关键核心知识点的 `BELONGS_TO` 关系已覆盖

---

## 输出规范

**最终向用户交付：**
1. `knowledge_graph_triples.csv` — 核心三元组文件（必须）
2. `knowledge_nodes.csv` — 知识点节点详情（推荐）
3. 简要的图谱统计摘要（在对话中说明）：
   - 共抽取知识点数量
   - 共生成三元组数量
   - 关系类型分布
   - 覆盖的章节/模块列表

---

## 多文件处理策略

当用户提供多种类型的教学资源时：

1. **分类读取**：按文件类型分批处理
2. **去重合并**：跨文件合并同一知识点（名称规范化后去重）
3. **标注来源**：`source` 字段记录每条三元组的来源文件
4. **权重叠加**：同一三元组在多个文件中出现时，`confidence` 提升为 `high`

---

## 典型用例示例

**用户输入：** "这是我们《机器学习》课程的教学大纲和两章PPT，帮我建立知识图谱"

**Claude 的处理步骤：**
1. 读取文件内容（大纲 + PPT）
2. 抽取知识点：`机器学习`、`监督学习`、`决策树`、`特征工程`……（共 N 个）
3. 建立关系：`决策树 IS_A 监督学习算法`、`特征工程 PREREQUISITE_OF 模型训练`……
4. 生成两个 CSV 文件并输出统计摘要

---

## 注意事项

- **专业术语保留**：领域专有名词保持原语言（如公式符号、英文缩写）
- **粒度控制**：单门课程的知识点建议在 50-300 个之间；过少说明抽取不够，过多说明粒度太细
- **循环依赖**：如发现 A PREREQUISITE_OF B 且 B PREREQUISITE_OF A 的情况，标注 `confidence=low` 并提示用户核查
- **大文件处理**：如文件超过 100 页/MB，分章节逐步处理，最后合并
